# Triggers

A trigger is an event source. The handler decides whether to start the agent.

Built-in triggers:

- Cron: `cron`
- GitHub: `github`
- Grafana: `grafana`
- Linear: `linear`
- Slack: `slack`
- Telegram: `telegram`
- Webhook: `webhook` (generic JSON receiver for sources without a preset)

```ts
import { cron, github } from 'roboport/triggers';

// Time-based, fires in-process
agent.on(cron({ schedule: { every: 'day', at: { hour: 9 } } }), async () => {
  await using session = agent.session();
  await session.send('Post the daily standup summary.');
});

// Webhook-based
const gh = github({ secret: process.env.GITHUB_WEBHOOK_SECRET });
agent.on(gh.pullRequest({ actions: ['opened'] }), async (event) => {
  await using session = agent.session();
  await session.send(`Review PR #${event.number}.`);
});

await agent.start();
```

A webhook trigger needs a URL to receive events. Bind the receiver's `handle` to a route:

```ts
app.post('/webhooks/github', (c) => gh.handle(c.req.raw));
```

`slack` is an Events API receiver — it verifies the request signature, answers
the setup challenge, and exposes `appMention`, `message`, and `reaction`:

```ts
import { slack } from 'roboport/triggers';

const sl = slack({ signingSecret: process.env.SLACK_SIGNING_SECRET });
agent.on(sl.appMention(), async (event) => {
  await using session = agent.session();
  await session.send(`Mentioned in ${event.channel}: ${event.text}`);
});

app.post('/webhooks/slack', (c) => sl.handle(c.req.raw));
```

For a source without a dedicated preset, `webhook` parses JSON deliveries into
`{ body, headers }`, with optional HMAC verification and id-based dedup:

```ts
import { webhook } from 'roboport/triggers';

const hook = webhook({ secret: process.env.WEBHOOK_SECRET });
agent.on(
  hook.event({ filter: (e) => (e.body as { kind: string }).kind === 'deploy' }),
  async ({ body }) => {
    await using session = agent.session();
    await session.send(`Deploy event: ${JSON.stringify(body)}`);
  },
);

app.post('/webhooks/custom', (c) => hook.handle(c.req.raw));
```

## Custom trigger

`trigger` builds one from a `start` function that receives an `emit` callback and returns an unsubscribe:

```ts
import { trigger } from 'roboport/triggers';

const tick = trigger<{ at: number }>({
  name: 'tick',
  start(emit) {
    const id = setInterval(() => emit({ at: Date.now() }), 60_000);
    return () => clearInterval(id);
  },
});

agent.on(tick, async (event) => {
  await using session = agent.session();
  await session.send(`Tick at ${event.at}.`);
});
```
