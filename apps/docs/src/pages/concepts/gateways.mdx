# Gateways

A gateway is a two-way connection between a chat transport (Telegram, Slack, …) and an agent. Where a [trigger](/concepts/triggers) is one-way (it fires an event and forgets), a gateway accepts inbound messages from the **transport** and uses a **reply channel** to relay the answer back.

Built-in gateways:

- Telegram: `telegramGateway`

Use `serve` to bind an agent to a gateway:

```ts
import { serve, telegramGateway } from 'roboport/gateways';

const bot = serve(
  agent,
  telegramGateway({ token: process.env.TELEGRAM_TOKEN }),
);
// later: await bot.stop();
```

## Transports

Gateways support two transports.

By default, polling is used. Zero config, ideal for local development.

You can also use webhooks, which are recommended in production:

```ts
const gateway = telegramGateway({
  token: process.env.TELEGRAM_TOKEN,
  transport: {
    mode: 'webhook',
    secretToken: process.env.TELEGRAM_WEBHOOK_SECRET,
  },
});
const bot = serve(agent, gateway);

const route = '/webhooks/telegram';
app.post(route, (c) => bot.handle(c.req.raw));
await gateway.client.setWebhook(`${publicUrl}${route}`, {
  secretToken: process.env.TELEGRAM_WEBHOOK_SECRET,
  allowedUpdates: ['message'],
});
```

## Rich replies

The Telegram gateway sends replies as [rich messages](https://core.telegram.org/bots/api#rich-markdown-style): the reply text is treated as GitHub-Flavored Markdown and rendered with headings, lists, tables, blockquotes, fenced code blocks, math, and collapsible blocks — so an agent's Markdown output shows up formatted instead of as literal `**` and backticks. Streaming replies (`stream`) animate an ephemeral rich draft bubble while the turn runs, then commit the final rich message. No escaping or configuration is required.

## Custom gateway

A gateway implements the `Gateway` interface: `open(handler)` wires the transport and returns an unsubscribe (mirroring a trigger's `start`), `handle?(req)` is present for webhook transports the host mounts in an HTTP server, and `relay?` sets the default reply strategy. Each inbound message arrives with a `Channel` carrying `send(text)` and an optional `thinking()` presence indicator. `telegramGateway` is the reference implementation.
